Если коротко: **Cursor** — самый быстрый старт для тех, кто хочет «привычный VS Code, только с агентом внутри». **Claude Code** — выбор тех, кто работает в терминале, любит длинные автономные сессии и хочет максимально гибкие хуки (MCP, скрипты, кастом-команды). **Windsurf** — компромисс с упором на многошаговое планирование и обзор изменений.

Все три инструмента в 2026 году умеют примерно одно и то же: автономно править несколько файлов, запускать тесты и линтеры, читать логи, искать по репозиторию, работать с git. Разница — в эргономике, ценах, ограничениях контекста и в том, как именно агент принимает решения.

## Cursor: VS Code-форк с агентом

Cursor — форк VS Code, в который вшит чат с агентом и режим Composer (правка нескольких файлов одной задачей). По умолчанию работает с Claude Sonnet/Opus и GPT-серии, можно подключать собственные API-ключи.

Плюсы:

- **Низкий порог входа.** Если у вас уже VS Code — переезд занимает минут пять, все плагины, темы, шорткаты на месте.
- **Быстрый автокомплит.** Tab-автодополнение часто лучше, чем у GitHub Copilot — Cursor видит свежие правки и продолжает мысль.
- **@-команды.** Можно подсовывать в чат конкретные файлы, символы, документацию (`@Docs`), git-историю.
- **Composer.** Режим, где агент сразу правит несколько файлов и показывает diff целиком — удобно для рефакторов.

Минусы:

- В долгой автономной сессии (час+) Cursor чаще теряет нить задачи, чем CLI-агенты.
- Тарифы перешли на pay-per-token — на тяжёлых моделях счёт может расти быстрее, чем кажется.
- Ограничения контекста: иногда агент «забывает» файлы, которые сам только что редактировал, если их нет в активной вкладке.

Кому подходит: разработчикам, которые редактируют код руками 60–70% времени, а AI используют как ускоритель — генерацию тестов, рефакторинг, разбор незнакомого куска.

## Claude Code: терминал и автономия

Claude Code — официальный CLI от Anthropic. Запускается как `claude` в терминале внутри директории проекта и работает с любой IDE по соседству — VS Code, Vim, JetBrains. У него нет своего редактора: он читает и пишет файлы напрямую, а вы видите изменения в diff-формате или через git.

Плюсы:

- **Длинные автономные сессии.** Claude Code заточен под задачи на 30–60 минут и больше: «реализуй фичу, напиши тесты, прогони, поправь, открой PR».
- **MCP-серверы.** Стандартный механизм подключения внешних сервисов — БД, GitHub, Slack, документация, кастомные скрипты. На 2026 год это самый зрелый протокол.
- **Кастомные slash-команды и хуки.** Можно описать словами, что делать перед/после редактирования, какие скрипты запускать.
- **Sub-agents.** Параллельные дочерние агенты для исследования кодовой базы или нескольких задач сразу.
- **Цена прозрачнее.** Тариф Claude Max включает фиксированное окно использования; за пределами — pay-per-token.

Минусы:

- Нет визуального IDE — diff и git-merges смотришь в терминале или внешнем редакторе.
- Кривая входа выше: чтобы выжать максимум, нужно описать команды, MCP-конфиг, договориться с собой о структуре сессий.
- Работает только с моделями Anthropic — нельзя «переключиться на GPT» нажатием кнопки.

Кому подходит: backend-разработчикам, девопсам, инфра-инженерам — всем, кто и так живёт в терминале. Тем, кто строит автоматизацию вокруг кода: CI-скрипты, миграции, генерация. Тем, кому нужны автономные многочасовые задачи без надзора.

## Windsurf: IDE с планированием

Windsurf — агентный IDE от Codeium, тоже основан на VS Code. Главная фишка — Cascade, режим, в котором агент сначала строит план изменений, потом исполняет его шаг за шагом, показывая прогресс.

Плюсы:

- **Прозрачное планирование.** Прежде чем что-то править, агент показывает, какие файлы тронет и что в них сделает — легче перехватить, если он пошёл не туда.
- **Memories.** Агент копит знания о вашем проекте (стиль, соглашения, что не трогать) между сессиями — без явного `.md` файла.
- **Tab-автокомплит от Codeium.** Один из лучших на рынке по скорости и попадаемости.
- **Гибкие тарифы.** Есть бесплатный план для одиночных разработчиков, корпоративные планы — без жёсткого pay-per-token.

Минусы:

- Меньше плагинов и тем, чем у Cursor (хотя совместимость с VS Code есть).
- MCP-экосистема пока тоньше, чем у Claude Code.
- В сложных задачах планирование иногда мешает: агент тратит время на детализацию там, где быстрее было бы просто сделать.

Кому подходит: разработчикам, которым важен контроль и предсказуемость — увидеть план до того, как агент начнёт ломать файлы. Командам, которые подбирают единый инструмент с понятным ценником.

## Сравнительная таблица

Свести всё в одну таблицу полезно, но помните: в агентных IDE главное не «список фич», а ощущение от часа реальной работы. Это та область, где надо попробовать самому.

- **Тип:** Cursor — IDE; Claude Code — CLI; Windsurf — IDE.
- **Базовая модель:** Cursor — Claude + GPT; Claude Code — только Claude; Windsurf — Claude + GPT + собственные.
- **Сильная сторона:** Cursor — быстрый старт и автокомплит; Claude Code — длинные сессии и MCP; Windsurf — планирование и memories.
- **Ценник на одиночного разработчика:** Cursor — pay-per-token, ~$20–40/мес типично; Claude Code — Max-подписка $20–100/мес плюс перерасход; Windsurf — есть free-tier.
- **Кривая обучения:** Cursor — низкая; Claude Code — средняя; Windsurf — низкая–средняя.

## Когда что брать: четыре сценария

### 1. Вы фронтендер на VS Code и хотите ускорить рутину

Берите Cursor. Композер для рефакторов компонентов, Tab для автодополнения и `@`-команды для подсовывания документации компонента-библиотеки — этого хватит, чтобы экономить часы в день. Claude Code тут избыточен: визуальная среда важнее автономии.

### 2. Вы делаете автономные ночные миграции на бэкенде

Claude Code. Опишите задачу, дайте инструменты через MCP (доступ к БД на staging, доступ к GitHub PR API), и пусть он сам ходит по репозиторию, ставит миграции, гоняет тесты. Cursor и Windsurf для такого хуже — слишком много визуального шума, и сессии у IDE-агентов короче.

### 3. Вы тимлид и подбираете инструмент команде из 10 человек

Windsurf — самый сбалансированный выбор для команд: предсказуемый ценник, корпоративные SSO/политики, режим планирования помогает джунам не сломать прод. Cursor — тоже годится, но pay-per-token делает биллинг непрозрачным. Claude Code для команды джунов слишком требователен.

### 4. Вы пробуете вайбкодинг впервые

Cursor — самый низкий порог. Через 10 минут вы уже что-то делаете. Потом, освоившись, можно попробовать Claude Code и посмотреть, насколько вам зайдёт стиль «дать задачу и уйти на час».

## Что они НЕ умеют (хорошо)

Полезно держать в голове общие ограничения, которые в 2026 году ещё актуальны для всех трёх инструментов.

- Сложный архитектурный дизайн с нуля — все три агента делают типовые шаблоны, но абстракции под ваш домен предложат шаблонные.
- UI-вкус — генерируемые интерфейсы получаются «AI-эстетика»: одинаково правильные и одинаково скучные. Полировка дизайна — на вас.
- Длинные многошаговые задачи без явного промежуточного контроля. Даже Claude Code без подсказок может улететь в попытку решить не ту задачу.
- Работа с моно-репо размером в гигабайты — все агенты упираются в окно контекста, нужны грамотные индексы и `.cursorrules` / `CLAUDE.md`.

## Минимальный набор для эффективного вайбкодинга

Независимо от выбранного инструмента, чтобы агент перестал тратить ваше время впустую, в репозитории должны быть:

1. Файл с инструкциями для агента (`CLAUDE.md`, `.cursorrules`, `windsurfrules` — у каждого свой) с описанием стиля кода, ключевых файлов, того, что НЕ трогать.
2. Рабочие тесты и линтер — агент должен иметь возможность сам проверить себя.
3. Чёткие команды запуска: `npm run dev`, `make test`, `docker compose up` — без них агент тратит много шагов на угадывание.
4. Описание архитектуры в README или ADR — иначе агент придумает свою.
